# Frontend (src/frontend/)

## Overview

React app for visualizing Claude Code session logs. Renders an interactive timeline with agent hierarchies and event details.

## Entry Points

- `index.html` ‚Äî HTML entry point that imports the React app
- `frontend.tsx` ‚Äî React DOM mounting with `QueryClientProvider` and `RouterProvider`
- `routes/__root.tsx` ‚Äî Root layout (dark background shell with `<Outlet />`)
- `routes/index.tsx` ‚Äî Home route (session browser + timeline, type-safe search params)

## Features

1. **Directory/Session selector** ‚Äî Dropdown UI in `Header` to browse and load sessions, with URL persistence (`?directory=<dir>&session=<path>`) via TanStack Router search params
2. **GraphTimeline** ‚Äî Main visualization showing events chronologically with agent context
3. **EventDetailsPanel** ‚Äî Side panel displaying full event data when selected

## Files

- `frontend.tsx` ‚Äî React DOM entry: QueryClient + router instance + RouterProvider
- `routes/__root.tsx` ‚Äî Root layout component (wraps all pages)
- `routes/index.tsx` ‚Äî Home route with search param validation (`directory`, `session`)
- `routeTree.gen.ts` ‚Äî Auto-generated by `@tanstack/router-cli` (do not edit)
- `api.ts` ‚Äî TanStack Query hooks and fetch helper
- `store.ts` ‚Äî Zustand store for transient UI state (`selectedEvent` only)
- `index.css` ‚Äî Styles
- `components/Header.tsx` ‚Äî Directory/session selector dropdowns (reads/writes search params)
- `components/GraphTimeline.tsx` ‚Äî Timeline visualization
- `components/EventDetailsPanel.tsx` ‚Äî Event detail side panel
- `components/EventList.tsx` ‚Äî Event list within timeline
- `components/AgentTree.tsx` ‚Äî Agent hierarchy tree view
- `components/MarkdownContent.tsx` ‚Äî Markdown rendering

## Routing

Uses `@tanstack/react-router` with file-based routing via `@tanstack/router-cli`.

The router is configured in `frontend.tsx`. Route files live in `routes/`. The generated route tree (`routeTree.gen.ts`) is committed and regenerated via `bun run routes:generate`.

### Search params

The index route validates search params with Zod:

```ts
const searchSchema = z.object({
  directory: z.optional(z.string()).catch(undefined),
  session: z.optional(z.string()).catch(undefined),
})
```

Navigation uses `useNavigate` from TanStack Router. `Header` reads params via `Route.useSearch()`.

## Data Fetching

Uses TanStack Query (`@tanstack/react-query`) for all server communication. Query/mutation hooks are defined in `api.ts`. CLI-provided sessions are handled by a `useCliSession` hook that attempts to load `/api/session` without parameters.

## State

- **URL search params** ‚Äî source of truth for `directory` and `session` selection (TanStack Router)
- **Zustand** (`store.ts`) ‚Äî transient UI state only: `selectedEvent` (which event's detail panel is open)
